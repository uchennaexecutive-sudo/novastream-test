Meet the Cast

Every file, every actor, and exactly what job they do

Your production crew

Before you can direct a film production, you need to know who does what. Same here — Nova Stream has about 15 key "characters", each with a very specific job. Knowing them means you can tell an AI assistant exactly where to make changes instead of hoping it guesses right.

Think of the codebase as a film set. Every department has a clear role and nobody steps on anyone else's toes.

🎬
Pages

The sets — full-screen views the user navigates between. Each page is one React component in src/pages/.

e.g. Home.jsx, Detail.jsx

🎭
Components

The props & costumes — reusable UI pieces that appear across multiple pages. Live in src/components/.

e.g. MoviePlayer.jsx, Sidebar.jsx

📓
Stores

The production diary — the app's shared memory. Any page can read or update these. Powered by Zustand.

e.g. useAppStore.js, useAuthStore.js

🔬
Lib

The specialist consultants — helper files that talk to external services. One file per service in src/lib/.

e.g. tmdb.js, anilist.js, progress.js

⚙️
Rust Commands

The special effects department — does things JavaScript cannot: bypassing CORS, opening hidden browser windows, streaming files to disk.

All live in src-tauri/src/main.rs

💡
The golden rule: one file, one job

tmdb.js only talks to TMDB. MoviePlayer.jsx only renders the player. useAppStore.js only manages app-level state. This means when something breaks, you know which file to open.

Reading real code — the Zustand store

Here is what a Zustand store looks like in practice. This one handles the app's theme — and shows the write-through sync pattern in one elegant block.

useAppStore.js import { create } from 'zustand' import { syncProfileSetting } from '../lib/supabase' const useAppStore = create((set, get) => ({ theme: localStorage.getItem('nova-theme') || 'nova-dark', setTheme: (id) => { document.documentElement .setAttribute('data-theme', id) localStorage.setItem('nova-theme', id) set({ theme: id }) syncProfileSetting({ theme: id }) },
Plain English

Line 1: Import Zustand — the shared memory manager

Line 2: Import the helper that syncs settings to the cloud

Line 4: Create a store — a shared object the whole app can read and update

Line 5–6: The current theme is read from localStorage so it survives app restarts

Lines 8–14: When someone changes the theme: update the DOM immediately → save to localStorage → update the store → sync to cloud. All four steps in one action.

🧠
The Zustand pattern

State lives in one place. Actions like setTheme update it everywhere at once — React components, localStorage, and the cloud. No tangled prop-passing across layers of components.

The bridge between JavaScript and Rust

React runs inside a browser layer. Rust runs natively on the OS. The only way they communicate is through a function called invoke(). Here is what that looks like in src/lib/movieStreams.js.

movieStreams.js import { invoke } from '@tauri-apps/api/core' async function fetchResolvedStreams( tmdbId, contentType, season, episode, imdbId, options = {} ) { return invoke('fetch_movie_resolver_streams', { payload: { tmdbId: String(tmdbId), contentType, season: season ?? null, episode: episode ?? null, imdbId: imdbId || null, excludeUrls: Array.isArray( options.excludeUrls ) ? options.excludeUrls : [], }, }) }
Plain English

Line 1: Import invoke() — the magic doorbell that rings into Rust

Line 3–6: A JavaScript function that accepts movie details as inputs

Line 7: invoke() is the single line that crosses from JavaScript into Rust — everything after this is handled natively

Lines 8–15: Package the movie details into a structured payload Rust can understand — IDs, type, season, episode number

Result: Rust queries the Nuvio sidecar, validates stream URLs, and returns the best candidates as a JavaScript Promise

Why this matters when making changes

If you want to change what is displayed, edit a Page. If you want to change the player UI, edit a Component. If you want to change what the app remembers, edit a Store. If you want to change a network call, edit a Lib file. If you need native capabilities, the answer is always Rust.
