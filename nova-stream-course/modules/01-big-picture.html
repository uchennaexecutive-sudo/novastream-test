The Big Picture

What Nova Stream is, how it’s built, and what happens when you press Play

Opening Hook

You just pressed Play on a movie. In the next 3 seconds, 4 completely different systems fired up — a React UI, a Tauri shell, a Rust engine, and a local streaming server. Here’s how they all work together.

What is Nova Stream?

Nova Stream is a premium streaming desktop application for Windows and macOS. It looks and feels like Netflix, but runs as a native app installed on your computer. You browse, you press Play, and a video starts — no browser tab, no Flash plugin, no web dependency.

🎬

Movies

Browse, search, and watch feature films. Streams are resolved on the fly by a local server running on your machine, not a remote CDN.

📺

Series

Full episode navigation, season switching, and a resume system that remembers exactly where you left off — per episode, not just per show.

⛩️

Anime

Powered by AniList with its own separate playback engine and scraping logic that’s isolated from the movie/series path.

✨

Animation

Uses the same stream resolver as movies but with a tuned provider preference so it doesn’t compete with movie-specific fast paths.

💻
It’s a real app, not a website

Nova Stream can access your file system, run native code, download updates silently, and make network requests that a web browser would refuse. This is the key difference — and it changes everything about how it’s built.

Three Layers of the App

Think of it like a spaceship. The cockpit (React) is what the pilot touches. Mission control (Tauri) is the bridge that makes the ship real. The engines (Rust) do the propulsion no human could do by hand.

Layer 1 — What You See
🎨

React + Zustand + Framer Motion

Every button, card, sidebar, player control, and page you see. React draws the UI. Zustand remembers state (watchlist, theme, search open/closed). Framer Motion makes it move smoothly.

Pages (Home, Detail, Movies…) Components (Sidebar, Player, Search) Zustand Stores Vite build system TailwindCSS styles
⚡ invoke() — the magic border
Layer 2 — The Bridge
🌉

Tauri 2 Bridge

Wraps React in a real Windows/macOS app window. Most importantly, it opens a two-way channel — called invoke() — that lets JavaScript call Rust functions as if they were just regular code.

🔧 native OS calls & file access
Layer 3 — The Engine
⚙️

Rust Engine

Does the heavy lifting that a browser literally cannot: fetches video streams that block web requests, downloads 50MB updates without freezing the UI, manages scraping sessions, and runs the Nuvio sidecar server.

Stream resolver HLS manifest fetcher Update downloader Session manager Nuvio sidecar manager
🔩
Why Rust? The honest answer.

Some things — like grabbing a video stream that blocks browsers, or downloading a 50MB update without freezing the UI — need to run in a native process, not inside a web tab. Rust is the language that makes that possible while keeping the app safe and fast. CORS restrictions that would stop a web app cold are completely irrelevant to Rust.

External services in the picture

Nova Stream also calls out to four external services: TMDB for movie metadata, AniList for anime data, Nuvio for stream resolution, and Supabase for optional user account sync. The app works without an account; Supabase is only used when you sign in.
